# Contributing to OpenDT

Thank you for wanting to contribute to OpenDT! You can contribute in various ways:

* Report a bug through [GitHub issues](https://github.com/atlarge-research/opendt/issues).
* Propose new functionality for using this project.
* Contribute improvements to the code and documentation.
* Provide feedback about how we can improve the project.
* Help answer questions on our [Discussions](https://github.com/atlarge-research/opendt/discussions) page.

## Want to report a bug or suggest a feature?

Please file an issue! First, check if someone has already filed an issue addressing your concern. If there is an existing issue, feel free to comment to show your support or add additional information. You can also react with a thumbs-up to help us prioritize.

If you can't find an existing issue, open a new one. Describe actual and expected behavior in detail.

## Want to contribute code?

Fork the [repository](https://github.com/atlarge-research/opendt) and submit a pull request when you're ready. Describe *what* you changed and *why* to help us understand your contribution.

Please follow common Git standards when writing commit messages. See [this post](https://cbea.ms/git-commit/) for details.

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/opendt.git
cd opendt

# Set up development environment
make setup

# Run tests
make test
```
